## 1. 개요

- **서비스 유형:** 팬덤 서비스 (게이미피케이션 적용)
- **플랫폼:** LINE LIFF (LINE 미니앱)
- **한 줄 정의:** 팬이 매일 10초 행동하고, 그 행동을 응모권으로 전환하는 팬덤 습관 엔진
- **핵심 재미:** 수집(카드) + 보상(응모권 → 이벤트 당첨 기대) + 소속감(광장 채팅) + 창작(팬 섬 꾸미기)
- **UX 철학:** 생각 없이, 하루 10초, 버튼 3번, 반복

## 2. 코어 루프

```
유입 (X / LINE / 외부)
  → 오시 선택 (최초 1회)
  → 응원하기 (하루 1회) → 응모권 +1
  → Today 카드 받기 (하루 1회) → 응모권 +1
  → 광장 글쓰기 (하루 1회) → 응모권 +1 [후보]
  → 팬 섬 꾸미기 (자유 시간)
  → 이벤트 응모
  → LINE 알림 (당첨 / 리마인드)
  → 재접속
```

- **핵심 세션 흐름:** 접속 → 응원하기 → Today 카드 받기 → (섬 꾸미기) → 이탈
- **예상 세션 시간:** 10초~1분 (기본) / 섬 꾸미기 시 5~10분

## 3. 기능 시스템

### 3-1. 오시(최애) 선택

- 최초 접속 시 5명 멤버 중 1명 선택 (온보딩)
- 홈 UI가 선택한 오시 중심으로 개인화 (캐릭터, 컬러 테마)
- 이후 변경 가능 (횟수 제한 없음)

---

### 3-2. 응원 시스템

- 하루 1회 응원 버튼 탭 → 응모권 1개 획득
- 결과 피드백: +1 애니메이션
- 연속 출석 콤보: N일 연속 응원 시 보너스 응모권 (N 수치 추후 확정)
- 개인 기록만 표시 — 멤버 간 랭킹 없음, 경쟁 없음

---

### 3-3. Today 카드 시스템

- 하루 1회 카드 지급 (무작위 등급)
- **등급 확률:** Normal 60% / Rare 30% / Special 10%
- **카드 구성 요소:** chibi 캐릭터 + 멤버명 + 오늘의 메시지 + 날짜
- "오늘만 획득 가능" 한정성 강조
- 카드 획득 시 응모권 1개 추가 지급
- 카드 획득 연출: 뒤집기 애니메이션

---

### 3-4. 카드북 (컬렉션)

- 멤버별 탭 5개
- 획득 카드: 컬러 표시 / 미획득 카드: 실루엣 + 자물쇠 아이콘
- 멤버별 컬렉션 진행률 표시 (예: 8/20)
- **카드 총 구성:** 5명 × 20장 = 100장
  - Normal 12장 / Rare 6장 / Special 2장 (멤버당)
- 기존 chibi 일러스트 5장을 스타일 앵커로 활용, 포즈/의상 변형으로 카드 생성

---

### 3-5. 광장 채팅

- **상단 고정 레이어:** 멤버 공식 포스트 (운영팀 등록, 항상 표시)
- **하단 팬 타임라인:** 비동기 텍스트 게시 (최신순 스크롤)
- 다국어 입력 허용 (어떤 언어로 써도 무방)
- 번역 기능: MVP 제외, 2단계에서 추가
- 팬 글쓰기: 텍스트만 (이미지 첨부 MVP 제외)
- 신고 / 차단 기능 포함 (운영 필수)

---

### 3-6. 이벤트 & 응모권

**이벤트 2트랙:**

| 트랙 | 상품 | 운영 시기 |
|------|------|----------|
| 상시 트랙 | 앱 전용 스페셜 카드, 오시 이모티콘 팩 | 항상 (월 1~2회 교체) |
| 스페셜 트랙 | 팬미팅 / 콘서트 초청 | 일본 활동 시에만 |

- 응모권 투입 → 추첨 → LINE 알림으로 당첨 결과 안내
- 포토카드 상품 미사용

### 3-7. 팬 섬 (아이소메트릭 꾸미기)

**개요:**
각 유저가 자신만의 아이소메트릭 공간(섬)을 꾸미는 기능. 오시 chibi 캐릭터가 섬에 상주하며, 데코 아이템 구매가 핵심 BM과 연결됨.

**기술 기반:**
기존 완성된 아이소메트릭 엔진 활용 (`diy_land_simple.html` → React 컴포넌트로 포팅)

**구성:**
- 아이소메트릭 타일 그리드 (다이아몬드 타일, green/white/pink 3종 기본 제공)
- 데코 아이템 배치 (팔레트에서 선택 → 타일 위 배치)
- 오시 chibi 캐릭터 상주 (선택한 오시가 섬 안을 돌아다님)
- 저장 → 광장에서 내 섬 공유 가능

**BM 연결:**
| 아이템 | 가격대 | 응모권 지급 |
|--------|--------|------------|
| 기본 데코 소품 | ¥100~200 | 1개 |
| 오시 테마 세트 | ¥500~800 | 3개 |
| 시즌 한정 데코 | ¥300~500 | 2개 |

- 데코 아이템 구매 = 실질 상품 판매 → 응모권은 구매 보너스
- 무료 응모권 경로(응원+카드) 병행 → 오픈 현상 구조 유지 (일본 경품표시법 준수)
- 아이템은 영구 소유 (기간 제한 없음) — K-pop 팬 굿즈 소장 문화 반영

**콘텐츠 업데이트:**
- 시즌별 테마 세트 (봄/여름/가을/겨울)
- 이벤트 한정 데코 (팬미팅 기념 등)
- 오시별 시그니처 가구

## 4. 경제 시스템 (응모권)

### 획득 구조

| 행동 | 응모권 | 비고 |
|------|--------|------|
| 응원하기 | 1개 | 하루 1회, 무료 |
| Today 카드 받기 | 1개 | 하루 1회, 무료 |
| 광장 글쓰기 | 1개 | 하루 1회, 무료 [후보] |
| 기본 데코 구매 | 1개 | 구매 보너스 |
| 오시 테마 세트 구매 | 3개 | 구매 보너스 |
| 시즌 한정 데코 구매 | 2개 | 구매 보너스 |

### BM 구조

```
[무료 루프]
행동(응원+카드+광장) → 응모권 최대 3개/일

[유료 루프]
팬 섬 데코 구매 → 섬 꾸미기 + 응모권 보너스 지급

[응모]
응모권 → 이벤트 응모 → 당첨 기대
```

- **핵심 개념:** "상품(데코 아이템)을 팔고, 응모 기회를 덤으로 준다"
- **법적 구조:** 오픈 현상 (무료 경로 항상 존재 → 일본 경품표시법 상한 규정 미적용)
- 유료 응모권 단독 판매 금지 — 반드시 데코 아이템과 묶어서 판매


### 핵심 화면 플로우

```
첫 접속
  → 온보딩 (오시 선택)
  → 홈

홈
  → [응원하기] → 응원 완료 애니메이션
  → [Today 카드 받기] → 카드 획득 팝업 → 카드북 연결
  → 이벤트 배너 탭 → 이벤트 화면

프로필
  → [내 섬 꾸미기] → 아이소메트릭 에디터
       → 보유 아이템 팔레트
       → 타일/데코 배치
       → 저장 → 광장 공유 가능
  → [아이템 상점] → 데코 구매 → 응모권 지급 팝업

이벤트 화면
  → 응모권 투입 → 응모 완료 팝업
```

### 핵심 인터랙션

- **응원하기:** 탭 1회 → +1 애니메이션 → 응모권 획득 표시
- **Today 카드:** 탭 → 카드 뒤집기 연출 → 카드북 자동 저장
- **광장 글쓰기:** 하단 입력창 탭 → 텍스트 입력 → 게시
- **팬 섬 꾸미기:** 아이템 선택 → 타일 탭 → 배치 → 저장

## 8. 기술 요구사항

- **기술 스택:** LINE LIFF SDK 2.x + React 18 + TypeScript + Supabase + LINE Login
- **배포 타겟:** LINE 미니앱 (LIFF)
- **플랫폼 특이사항:**
  - iOS Safari 기반 LIFF 브라우저 → CSS `position:fixed` 주의
  - 뒤로가기 LINE 이탈 방지 → SPA 라우팅 (window.history 기반) 필수
  - 가챠 확률 로직은 서버사이드 처리 필수 (클라이언트 조작 방지)
- **아이소메트릭 엔진:** 기존 `diy_land_simple.html` → React 컴포넌트 포팅
  - 위치: `F:\Personal_project\brown_farm_ground_edit\resource\diy_land_simple\`
  - 기존 에셋(LINE Friends 브랜드) → KiiKii 스타일로 교체 필요
- **해상도:** 모바일 세로 고정 (360~430px 기준)
- **에셋 형식:** PNG (chibi 캐릭터, 카드 프레임, 아이소메트릭 데코)
- **사운드:** MVP 제외
- **다국어:** MVP는 한국어/일본어 UI, 번역 API는 2단계
- **인증:** LINE Login → LINE UID를 유저 PK로 사용
- **결제:** LINE Pay 연동 — 2단계 (MVP는 무료 루프 검증 우선)
- **알림:** LINE Messaging API — 2단계
- **지표:** 추후 확정
